# Amazon AMCとは？

![Amazon Marketing Cloud AMC](/images/blog/amazon-marketing-cloud-amc.jpeg)

[Amazon Marketing Cloud (AMC)](https://www.revns.com/blog/post/amazon-amc-vs-dsp-audience)は、Amazon Advertising専用に設計された高度な[分析ツール](https://www.revns.com/blog/post/master-amazon-seller-analytics)です。広告主は大規模なデータセットを探索し、オーディエンスの行動に関する貴重なインサイトを獲得し、複数のチャネルにわたってキャンペーンパフォーマンスを最適化することができます。マーケティング活動に包括的な可視性を提供するデジタル広告のコマンドセンターとしてお考えください。

## Amazon Marketing Cloudのドキュメンテーション

AMCを効果的に活用するには、主要なコンポーネントを理解する必要があります：

### データ統合と分析
Amazon Marketing Cloud (AMC)は、広告主がキャンペーンパフォーマンスを分析し、オーディエンスの行動を理解し、広告戦略を最適化できる、安全なクラウドベースの「クリーンルーム」環境です。これは、厳格なプライバシーとセキュリティ基準を維持しながら、複数の当事者がデータを共有して分析できる安全な空間を提供することで実現されています。この環境内で、異なるソースからのデータは匿名化され集計された方法で組み合わせることができ、個人を特定できる情報（PII）が露出または共有されることはありません。

AMCは以下の方法で広告主をサポートします：
*   **ファーストパーティデータのアップロードと統合** AMCは、広告主が自社のファーストパーティデータをAmazonの広告データと統合することを可能にします。このデータはAMC環境にアップロードされ、匿名化および集計された方法で処理されます。AMCは個々のユーザーデータが露出されないよう、厳格なデータプライバシー基準を維持します。
*   **クロスチャネル分析へのアクセス** AMCは、ブランドがクロスチャネルのキャンペーンパフォーマンスを分析し、オーディエンスの行動を理解し、広告を最適化することを可能にします。広告主とAmazon Advertisingの両方のデータを含む複数のデータセットにわたるイベントレベルのデータについて、カスタマイズ可能なレポートを提供します。これにより、キャンペーンパフォーマンスのより包括的な視点が得られます。
*   **顧客プライバシーコンプライアンスの維持** AMCは、購入者が個別に識別されることを防ぎ、分析対象となる購入者の個人識別情報（PII）へのアクセスを許可しません。各購入者は、購入者のプライバシーを保護するように設計された匿名化された「user_id」コードのシリーズによって識別されます。
*   **複雑なデータセットからの実用的なインサイトの生成** AMCは、広告主とそのパートナーのためのクリーンルーム環境を作成するために必要なインフラストラクチャを提供します。AMCのクリーンルーム環境内で、広告主はSQLベースのカスタムクエリを実行してデータセットを分析し、インサイトを発見し、キャンペーンを最適化することができます。結果は常に集計され、特定のユーザー情報を明らかにすることはできず、データのセキュリティとプライバシーコンプライアンスを確保します。

### キャンペーン最適化のための主要機能

1.  **高度なオーディエンスセグメンテーション**
    *   ブラウジングパターン、購入頻度、広告インタラクションに基づいてカスタムオーディエンスセグメントを作成
    *   プラットフォーム全体での購入行動と広告インタラクションを分析してターゲティング戦略を改善
    *   広告の関連性を高めコンバージョンを増加させるための的確なメッセージング戦略の開発

2.  **クロスチャネル分析**
    *   Amazon.co.jp、Fire TV、Alexaなどのアマゾンプラットフォーム全体での顧客インタラクションのモニタリング
    *   効果的なタッチポイントを特定するための顧客ジャーニーの理解
    *   クロスチャネルデータに基づく広告配置とタイミングの最適化によるキャンペーンROIの最大化

3.  **SQLベースのカスタムレポーティング**
    *   SQLクエリを使用してビジネス目標に関連する特定のメトリクスに焦点を当てたレポートのカスタマイズ
    *   コンバージョン率、広告リーチ、特定の広告グループに起因する売上などのKPIのトラッキング
    *   最も注目を集める商品のリスティングを改善するためのASINレベルでのパフォーマンス分析
    *   パフォーマンスデータに基づくキャンペーンのROI計算と予算配分の最適化

## Amazon Marketing Cloudの実際の応用例

### カート放棄の解決

Amazon Marketing Cloud (AMC)は、顧客が購入を完了せずに離脱する理由を分析するツールを提供することで、ブランドがカート放棄に対処することを可能にします。AMCを通じて、ブランドは放棄パターンに関するインサイトを得て、ターゲットを絞ったリターゲティングキャンペーンを作成し、顧客が戻って購入を完了するよう促すためのパーソナライズされたメッセージを開発することができます。AMCはまた、コンバージョン率を向上させるための最も効果的なインセンティブを決定するために、様々なプロモーション戦略をテストすることを可能にします。

### 広告頻度の最適化

AMCは、最適な広告露出頻度に関する重要なインサイトを提供し、ブランドが顧客がコンバージョンするまでに広告を何回見る必要があるかを理解するのを助けます。異なるチャネルにわたる顧客のコンバージョンパターンを分析することで、AMCは広告パフォーマンスの包括的な視点を提供します。これにより、チャネル固有のパフォーマンストラッキングが可能になり、コンバージョンを促進する最も効果的なチャネルを特定し、過剰露出と潜在的な顧客の離反を防ぐための広告疲労指標を提供します。

### ASINパフォーマンストラッキング

AMCは、個々のASIN（Amazon Standard Identification Numbers）のパフォーマンスを追跡することで、詳細な製品レベルの分析を可能にします。これにより、コンバージョン率の高い製品を特定し、ブランドが最も収益性の高いアイテムに広告努力を集中させることができます。AMCを使用することで、ブランドは最もコンバージョン可能性の高い製品にリソースを向けることで広告支出の配分を最適化し、より多くの顧客を引き付けるために製品リスティングを改善できる領域を特定することができます。

## Amazon AMCを使用するメリット

1.  **データ駆動型の意思決定**
    *   AMCは**包括的な分析**へのアクセスを提供し、広告主にキャンペーンとオーディエンスの行動についてのより良いインサイトを提供します。トレンドとパターンを分析することで、ユーザーは改善の機会を特定できます。
    *   AMCは**リアルタイムのパフォーマンストラッキング**ではなく遡及的な分析に焦点を当てていますが、広告キャンペーンの増分効果を測定し、オーガニックと広告による売上を区別することができます。ユーザーは即時の最適化のために他の[Amazonツール](https://www.revns.com/blog/post/amazon-fba--revenue-calculator-free)に頼ることができます。
    *   AMCは**実用的なインサイトの生成**を促進し、オーディエンスに響くキャンペーンを作成するのに役立ちます。AIコパイロットは、従来複雑だったAMC SQLの作成プロセスを簡素化し、データ分析パートナーとして機能します。
    *   データインサイトにより、ユーザーは事実に基づいた戦略的決定を下し、**エビデンスベースの戦略開発**につなげることができます。

2.  **キャンペーンパフォーマンスの向上**
    *   AMCの行動データはオーディエンスセグメンテーションを改善し、**ターゲティングの精度を向上**させます。ユーザーは行動と好みに基づいて特定の顧客セグメントにコンテンツと広告をカスタマイズできます。
    *   AMCは広告主がパフォーマンスの低いキャンペーンを特定して予算を再配分することを支援し、**より良い予算配分**につながります。どのチャネルが最高のパフォーマンスメトリクスを提供するかを評価することで、ユーザーはマーケティング予算の配分方法を決定できます。
    *   広告シーケンスを最適化しコンバージョンへの影響を測定することで、AMCは**より高いコンバージョン率**に貢献します。
    *   AMCのインサイトにより、広告主はパフォーマンスの高いキャンペーンに予算を再配分し、**広告支出収益率**（ROAS）を向上させることができます。AMCが可能にするクロスチャネル戦略により、ROASを25%改善することができます。

3.  **プライバシーに準拠した分析**
    *   AMCはデータを匿名化および集計することで**安全なデータ処理**を確保し、グローバルな規制に準拠し、分析のための安全な環境を作成します。機密情報が露出されることは決してありません。
    *   AMCは**匿名データ処理**を使用し、ユーザーレベルのデータを提供する代わりに少なくとも100ユーザーのグループの集計データを表示します。この集計により、ブランドはプライバシーを保護しながら顧客行動に関するインサイトを得ることができます。
    *   AMCはGDPR、CCPA、その他のグローバルなプライバシー規制を遵守し、**規制コンプライアンス**を確保します。
    *   AMCは顧客情報を保護します。匿名化された情報のみを受け入れ、ユーザーシグナルはAmazonによってエクスポートまたはアクセスされることはありません。ユーザーはAMCからの集計された匿名の出力にのみアクセスできます。

## Amazon Marketing Cloudを始めるには

### 前提条件
1.   **significant advertising spend（相当な広告支出）** 具体的な最低予算は普遍的に定められていませんが、一般的に年間5,000ドル以上の相当な広告予算が必要です。
2.   **技術的なSQL能力** AMC内でのデータのクエリと分析には、SQL（Structured Query Language）の習熟が不可欠ですが、Amazonは一般的なユースケース向けにあらかじめ構築されたクエリを提供しています。
3.   **robust first-party data（堅牢なファーストパーティデータ）** 高品質なファーストパーティデータの統合により、AMCから得られるインサイトが豊かになり、よりターゲットを絞った効果的な分析が可能になります。
4.   **データ駆動型マーケティングへのコミットメント** AMCのメリットを最大限に活用するには、データを活用してマーケティング戦略を形成しキャンペーンを最適化することへの献身が重要です。

### 実装ステップ
1.  **Amazonでの適格性の確認** Amazon DSPサービス契約が締結されているか、DSPを利用して過去28日以内にキャンペーンを実施 しているなど、組織が必要な基準を満たしていることを確認してください。ただし、最近のアップデートでは、すべてのAmazon広告主がAMCを利用できるようになったことが示唆されています。
2.  **Amazonの担当者に連絡** Amazonの担当者に連絡して、登録プロセスを開始し、具体的な要件について話し合います。
3.  **AMCダッシュボードの設定** データの可視化と分析のためにAMCダッシュボードを設定します。Amazon Web Services（AWS）アカウントが必要になる場合があります。
4.  **データ統合の設定** Amazon Adsキャンペーン、サードパーティプロバイダー、自社のファーストパーティデータなど、関連するデータソースを[API](https://www.revns.com/blog/post/amazon-advertising-api-documentation)、Event Manager、または統合シグナル管理ツールを通じて統合します。
5.  **SQLクエリの実行開始** SQLを使用して統合されたデータから実用的なインサイトを抽出し、キャンペーン戦術を最適化し、ビジネス上の意思決定を支援します。
6.  **キャンペーンのモニタリングと最適化** キャンペーンパフォーマンスを継続的にモニタリングし、AMCからのインサイトを活用してターゲティング、予算配分、クリエイティブ戦略を改善します。

## キャンペーン最適化戦略

### データ駆動型の調整

キャンペーンの効果を評価するために主要なパフォーマンスメトリクスをモニタリングします。データ内のトレンドとパターンを分析して改善の機会を特定します。広告クリエイティブやターゲティングパラメータの調整など、これらのインサイトに基づいて戦略的な変更を実施します。これらの変更の結果を継続的に追跡し、パフォーマンスを最大化しキャンペーン目標を達成するために戦略を繰り返し改善します。

### 高度なオーディエンスターゲティング

行動データを活用してオーディエンスの好みと行動をより深く理解します。デモグラフィック、興味関心、購入履歴に基づいてカスタムオーディエンスセグメントを作成します。エンゲージメントとコンバージョン率を向上させるために、各セグメントに響くターゲットメッセージを開発します。A/Bテストを通じてアプローチをテストし改善し、ターゲットオーディエンスにリーチし最適な結果を達成するための最も効果的な戦略を特定します。

### 予算最適化

パフォーマンスデータに基づいて広告支出を配分し、予算が効果的に使用されていることを確認します。最大の投資収益率を提供する高パフォーマンスのチャネルとキャンペーンに注力します。Return on Ad Spend（ROAS）に基づいて予算配分を調整し、収益性を最大化します。効率性とROIに焦点を当てながら、予算を増やしリーチを拡大することで成功しているキャンペーンをスケールアップします。

Amazon Marketing Cloudでの成功には、データ分析、テスト、最適化への継続的なコミットメントが必要であることを忘れないでください。上記の戦略に従い、アプローチを継続的に改善することで、AMCを活用して優れた広告結果を達成し、ブランドの持続可能な成長を促進することができます。

## よくある質問

### Q1: Amazon Marketing Cloud（AMC）にアクセスするための最低要件は何ですか？
A: AMCにアクセスするには、通常、Amazon Advertising プラットフォームでの相当な広告支出、チーム内の技術的なSQL能力、統合するための堅牢なファーストパーティデータが必要です。

### Q2: Amazon Marketing Cloudは在庫管理に役立ちますか？
A: AMCは主に広告分析に焦点を当てていますが、製品パフォーマンスと顧客需要パターンに関する詳細なデータを提供することで、間接的に在庫決定をサポートすることができます。

### Q3: AMCの使用にはSQLの知識が必須ですか？
A: はい、Amazon Marketing Cloudはデータ分析にSQLベースのクエリを使用するため、SQLの知識は不可欠です。ただし、一部のパートナーやエージェンシーがブランドのこの技術的な側面をサポートすることができます。

### Q4: AMCはどのようにデータプライバシーを確保していますか？
A: AMCは、すべての顧客データを匿名化し、プライバシー規制への厳格なコンプライアンスを維持しながら価値のあるインサイトを提供する、安全な「データクリーンルーム」環境で運用されています。
